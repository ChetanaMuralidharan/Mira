## Overview

MIRA (Medical Intelligence and Reasoning Agent) is an end-to-end clinical intelligence platform that enables users to ask healthcare analytics and clinical policy questions in natural language.

Instead of requiring users to write SQL or manually search clinical documentation, MIRA automatically determines how a question should be answered, retrieves the appropriate data, validates the results, generates visualizations when appropriate, and produces an explainable response.

Depending on the user's query, the system can reason over:

- **Structured clinical data** using Text-to-SQL
- **Clinical guidelines and policies** using Retrieval-Augmented Generation (RAG)
- **Both simultaneously** through a hybrid reasoning workflow

Unlike many AI portfolio projects that demonstrate Text-to-SQL, RAG, or agent frameworks independently, MIRA integrates modern **data engineering**, **analytics engineering**, **AI engineering**, **LLM orchestration**, **evaluation**, and **application development** into a single production-inspired architecture.

The project uses **synthetic patient data generated by Synthea**, ensuring realistic experimentation without exposing Protected Health Information (PHI).


# Engineering Highlights

One of the primary goals of MIRA was to build a system that demonstrates production-inspired AI engineering practices rather than isolated AI components.

Some notable architectural decisions include:

- Separation of structured analytics and unstructured knowledge retrieval
- Lakehouse-inspired analytical architecture
- Semantic schema retrieval instead of injecting the full database schema into prompts
- Hybrid vector and keyword retrieval using Reciprocal Rank Fusion
- MCP-based tool abstraction between the reasoning engine and backend services
- Automatic SQL validation and retry workflow
- Transparent reasoning with SQL, retrieved documents, validation status, and visualizations exposed to the user
- MLflow-based experiment tracking and benchmark evaluation
