# MIRA — Medical Intelligence and Reasoning Agent

> An autonomous clinical intelligence system that combines modern data engineering, retrieval-augmented generation (RAG), and agentic AI to answer clinical questions using natural language.

---

## Overview

**MIRA (Medical Intelligence and Reasoning Agent)** is a personal engineering project that explores how modern AI systems can reason across both structured clinical data and unstructured medical knowledge.

The project combines a lakehouse-style analytical warehouse, a semantic document retrieval system, and a LangGraph-based reasoning agent to answer questions such as:

* How many patients have Type 2 diabetes?
* What is the most common encounter class?
* What percentage of patients are female?
* What are the ADA recommendations for A1c targets in elderly patients?

Instead of treating SQL generation, retrieval-augmented generation (RAG), and data engineering as separate demonstrations, MIRA integrates them into a single end-to-end autonomous reasoning workflow.

The project uses **synthetic patient data generated by Synthea**, allowing realistic experimentation without exposing protected health information (PHI).

The goal of MIRA is to build a complete clinical intelligence system that resembles a production architecture while remaining fully local and reproducible.

Along the way, the project explores:

* Modern data engineering practices
* Autonomous reasoning workflows
* Clinical data modeling
* Retrieval-Augmented Generation
* Model Context Protocol (MCP)
* Explainable AI

# Engineering Notes

### Streamable HTTP MCP

The original implementation used MCP over stdio. During development, this exposed Windows-specific issues when communicating with DuckDB through long-running MCP tools.

The project was migrated to **Streamable HTTP MCP servers**, providing a more reliable architecture that also more closely resembles production deployments.

---

### Clinical Concept Mapping

While building the analytical warehouse, an important data-modeling issue was discovered.

The original transformation logic assumed ICD-10 diagnosis codes.

Synthea, however, exports conditions using **SNOMED-CT**.

The warehouse was redesigned to use explicit clinical concept mappings instead of ICD-10 pattern matching, significantly improving patient cohort identification.

---

### Explainability

Structured responses include:

* Generated SQL
* SQL validation
* Plain-English explanation
* Suggested follow-up question

Document-based responses include citations to the retrieved clinical sources.

# Running the Project

Clone the repository:

```bash
git clone https://github.com/ChetanaMuralidharan/Mira.git
cd Mira
```

Create a virtual environment:

```bash
python -m venv venv
```

Activate it (Windows):

```bash
venv\Scripts\activate
```

Install dependencies:

```bash
pip install -r requirements.txt
```

Start the required Docker services:

```bash
docker compose up -d
```

Start the MCP servers:

```bash
python start_servers.py
```

Run the LangGraph smoke test:

```bash
python -m agent.test_graph
```
